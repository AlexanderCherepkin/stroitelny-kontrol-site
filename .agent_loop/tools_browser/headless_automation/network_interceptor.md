# Browser Network Interceptor

## Role
Captures and filters XHR/fetch/resource traffic generated by the page, exposing useful request/response summaries while redacting credentials and oversized payloads.

## Contract

### Receives
- `session_handle`: from `session_manager.md`
- `filter`: optional substring/regex for URL filtering
- `resource_types`: list of types to capture (default `[xhr, fetch, document, script, stylesheet]`)
- `max_entries`: max captured entries (default 100)
- `include_response_body`: boolean — default false (privacy)

### Returns
- `captured_requests`: list of `{url, method, status, resource_type, request_headers, response_headers_summary, timestamp}`
- `redacted_count`: number of entries with sensitive headers removed
- `truncated`: boolean — whether entries exceeded `max_entries`

### Side effects
- Attaches route handlers to the browser context
- Logs capture event to `audit_logger.md`

## Decision Flow

1. **Resolve session and context** — return error if invalid.
2. **Install route** — use `context.route()` or `page.on('request')`/`page.on('response')` to intercept matching resource types.
3. **Filter and capture** — record URL, method, status, and headers for each matching request/response pair.
4. **Redact headers** — remove or mask `Authorization`, `Cookie`, `Set-Cookie`, `X-Api-Key`, and any header containing tokens.
5. **Truncate** — keep only the most recent `max_entries`; set `truncated=true` if more occurred.
6. **Return** — emit captured list and redaction counts.

## Failure Modes

| Condition | Response |
|---|---|
| Session invalid | Return error |
| Interception causes page breakage | Abort interception, return already-captured data, log to `error_handler.md` |
| Captured body too large | Omit body, include size and hash |
| Sensitive header slips through redaction | Re-scan and block entry if unredactable |
| HTTPS certificate error | Log warning, do not bypass unless explicitly allowed by `control/network_guard.md` |
