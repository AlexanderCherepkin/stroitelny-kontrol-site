# План создания отдельных страниц статей для сайта «Агентство строительного контроля»

## Контекст

Сайт — статический одностраничный лендинг в папке `site/`. Главная страница `index.html` содержит секцию «Статьи» с тремя карточками, ссылки которых ведут на `#` (заглушки). Пользователь требует сделать полноценные отдельные страницы для каждой статьи: длинные, на всю высоту/ширину контента, с фотографией из интернета, качественным оформлением и сохранением общего дизайна сайта.

## Цель

Создать три полноценные HTML-страницы статей, связанные с основным сайтом, с:
- единым header/footer/навигацией;
- большим содержательным текстом (4–5 разделов, практические советы, чек-листы, примеры);
- фотографиями по теме из открытых источников (Unsplash);
- адаптивной вёрсткой под десктоп, планшет и мобильные;
- ссылками «Читать далее» с главной страницы на новые страницы.

## Подход

1. **Стили**: создать новый файл `site/css/article.css` с компонентами:
   - `.article-hero` — компактный hero с фоновым изображением и заголовком;
   - `.article-meta` — дата публикации, время чтения, категория;
   - `.article-content` — основная колонка текста, растянутая на всю ширину контейнера, с удобочитаемой типографикой;
   - `.article-image` — полноширинные иллюстрации с подписями;
   - `.article-toc` — навигация по разделам статьи;
   - `.article-section` — разделы с подзаголовками, списками, выделенными блоками;
   - `.article-related` — блок «Другие статьи»;
   - `.article-back` — кнопка «← Назад к статьям».

2. **Страницы**: создать в `site/` три файла:
   - `article-why-construction-control.html` — «Зачем нужен строительный контроль?»
   - `article-gost-sp-snip.html` — «ГОСТ, СП и СНиП: что важно знать»
   - `article-how-to-choose-contractor.html` — «Как выбрать подрядчика»

   Каждая страница повторяет структуру `index.html`: header, footer, floating-actions, cookie-banner, модальные окна, подключение скриптов. Основное содержимое заменяется на статью.

3. **Изображения**: использовать прямые URL Unsplash с ключевыми словами по теме (construction site, building inspection, construction worker, blueprint). Для надёжности сохранить копии в `site/assets/images/articles/` через `ctx_fetch_and_index` или `WebFetch` → скачивание файла. Альтернатива: использовать внешние URL напрямую. Приоритет — скачать локально, чтобы страницы не ломались при отсутствии интернета.

4. **Обновление главной страницы**: в `site/index.html` заменить `href="#"` у трёх карточек статей на `href="article-why-construction-control.html"` и т.д.

5. **Верификация**: открыть каждую новую страницу в браузере через `http://localhost:8080/`, сделать скриншоты, проверить адаптивность.

## Порядок выполнения

1. Скачать 3–4 изображения по теме строительного контроля в `site/assets/images/articles/`.
2. Создать `site/css/article.css` с необходимыми компонентами.
3. Создать шаблон article base (в уме / в черновике) и на его основе сгенерировать три HTML-файла.
4. Написать/сгенерировать содержательный текст для каждой статьи (включая подзаголовки, списки, практические рекомендации, примеры).
5. Обновить ссылки в `site/index.html`.
6. Запустить/перезапустить локальный сервер и открыть все страницы в браузере.
7. Сделать скриншоты и проверить отсутствие ошибок.

## Валидация

- Все три страницы открываются по ссылкам с главной.
- Header/footer идентичны основному сайту.
- Текст не обрезается, читается комфортно.
- Изображения загружаются и корректно отображаются.
- Мобильная версия сохраняет читаемость (стек-верстка, масштабируемые изображения).
- Нет 404 при переходе по внутренним ссылкам.

## Риски и ограничения

- Внешние изображения могут быть недоступны; поэтому предпочтительно локальное сохранение.
- Длинный текст требует качественного наполнения; используем тематический контент без выдуманных конкретных цен/гарантий.
- Страницы статей не должны дублировать inline-CSS из `index.html` — используем внешние `site/css/*.css` + `article.css`, чтобы избежать дублирования.
