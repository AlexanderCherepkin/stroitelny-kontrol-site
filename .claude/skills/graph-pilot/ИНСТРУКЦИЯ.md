# 🧭 graph-pilot — как пользоваться

Скилл для Claude Code из видео про **Graphify**. Ты говоришь обычными словами, что хочешь
понять в коде — он сам подбирает нужную команду Graphify (а их больше сорока) и следит,
чтобы ты не потратил токены зря и не сломал Claude Code.

---

## Шаг 1. Положи скилл в ПРАВИЛЬНУЮ папку

⚠️ **Это главная ошибка.** Скилл нужно класть **строго** в папку `.claude/skills/` — не в
любую папку проекта. Если положишь куда попало (например, в папку с кодом или миграциями),
Claude его **не увидит**.

**Способ 1 — для всех проектов сразу (проще всего):**

Скопируй папку `graph-pilot` в свою домашнюю папку Claude:

```
Windows:  C:\Users\ТВОЙ_ЮЗЕР\.claude\skills\graph-pilot\
Mac:      ~/.claude/skills/graph-pilot/
```

**Способ 2 — только для одного проекта:**

Скопируй папку `graph-pilot` в корень проекта (туда, где лежат `package.json`, `CLAUDE.md`),
внутрь папки `.claude/skills/`:

```
<корень-проекта>/.claude/skills/graph-pilot/
```

### ✅ Как понять, что путь правильный

Должно получиться **именно так** (файл `SKILL.md` лежит внутри `graph-pilot`, а та — внутри `skills`):

```
.claude/
└── skills/
    └── graph-pilot/
        ├── SKILL.md       ← обязательно здесь
        ├── CHEATSHEET.md
        ├── README.md
        └── ИНСТРУКЦИЯ.md
```

### ❌ Частая ошибка

Положить папку в `src/`, `supabase/migrations/`, или просто в корень проекта **без**
`.claude/skills/`. Так Claude скилл не найдёт.

**После того как положил — перезапусти Claude Code** (закрой и открой заново).

---

## Шаг 2. Просто скажи, что хочешь

Открой Claude Code в своём проекте и формулируй задачу обычными словами:

> Разберись через graph-pilot, как здесь устроена авторизация.

> Я поменял много файлов — обнови карту проекта.

> Стоит ли вообще строить тут граф?

Скилл сам:
- поставит Graphify, если его нет;
- построит карту кода в **бесплатном** режиме;
- подберёт нужную команду под твой вопрос;
- не даст случайно включить платный режим там, где он не нужен;
- не станет ставить хук, который может сломать твой Claude Code.

---

## Шаг 3. Шпаргалка под рукой

Файл `CHEATSHEET.md` — это таблица «какая задача → какая команда Graphify». Держи открытой
или распечатай, если хочешь запускать команды руками.

---

## Когда Graphify тебе НЕ нужен

Если проект маленький (меньше сотни файлов) — граф почти не сэкономит, обычный поиск и так
быстрый. Скилл это понимает и переспросит, прежде чем строить граф. Граф реально окупается
на больших проектах — от пары сотен файлов и выше.

---

## Если не работает

| Что случилось | Что делать |
|---|---|
| Claude не видит скилл | Проверь, что файл лежит как `…/skills/graph-pilot/SKILL.md`, и перезапусти Claude Code |
| `graphify: команда не найдена` | Установи через `uv tool install graphifyy` (именно **graphifyy**, с двумя «y») |
| После установки хука Claude перестал читать файлы | Удали блок `hooks` со словом `graphify` из файла `~/.claude/settings.json`, перезапусти Claude Code |
| Граф «не экономит, как в видео» | Так и должно быть на маленьком проекте. Граф для больших — от 200+ файлов |
