Библиотека mem0ai/mem0 предоставляет масштабируемый гибридный слой долгосрочной памяти. Она берет на себя самое сложное: автоматически извлекает факты из диалога, векторизует их и сохраняет в базу (локальную Chroma/Qdrant или удаленную), а затем оценивает релевантность при поиске.Для твоей архитектуры это означает, что Mem0 может выступать единым разделяемым хранилищем контекста. Основной цикл и любые из управляющих субагентов (например, субагенты валидации или планирования) смогут независимо обращаться к этой памяти для принятия решений.Ниже подготовлен набор файлов для бесшовной интеграции этого слоя в твоего бота.1. Зависимости и конфигурацияСоздай файл requirements.txt для Python-окружения:Plaintextmem0aiopenaipython-dotenvchromadb(Примечание: Если часть твоих субагентов или сервисов работают на Node.js, Mem0 поддерживает и этот стек. Ты можешь установить клиент через npm install @mem0/mem0 и обращаться к той же базе).Создай файл .env для ключей API (Mem0 под капотом использует LLM для выделения фактов и создания эмбеддингов):Фрагмент кодаOPENAI_API_KEY=sk-твоя-копия-ключа# По желанию можно использовать свои модели или локальный Ollama2. Модуль управления памятью: memory_layer.pyЭтот файл выступает оберткой над Mem0. Его можно импортировать в любой субагент.Pythonimport osfrom dotenv import load_dotenvfrom mem0 import Memoryload_dotenv()class SharedMemoryLayer:    """    Изолированный слой памяти, к которому могут обращаться субагенты     для обогащения системных промптов актуальными фактами о пользователе.    """    def __init__(self):        # Настраиваем локальное хранение в ChromaDB.        # Это удобно для запуска внутри WSL или Docker-контейнеров.        config = {            "vector_store": {                "provider": "chroma",                "config": {                    "collection_name": "agent_memory_db",                    "path": "./memory_storage" # Данные будут сохраняться на диск                }            },            "llm": {                "provider": "openai",                "config": {                    "model": "gpt-4o-mini"                }            }        }        self.memory = Memory.from_config(config)    def store_cycle_context(self, user_message: str, agent_response: str, user_id: str):        """        Фоновый субагент сохранения.         Mem0 сам проанализирует этот массив и запишет только важные факты.        """        messages = [            {"role": "user", "content": user_message},            {"role": "assistant", "content": agent_response}        ]        self.memory.add(messages, user_id=user_id)    def fetch_relevant_facts(self, current_task_query: str, user_id: str) -> str:        """        Метод для основного цикла: достает факты, семантически близкие к текущему запросу.        """        results = self.memory.search(query=current_task_query, user_id=user_id)                if not results:            return "Нет сохраненных фактов."                    # results возвращает список словарей. Собираем их в удобную строку.        facts = [res["memory"] for res in results]        return "\n".join(f"- {fact}" for fact in facts)    def debug_memory(self, user_id: str):        """Вспомогательный метод для проверки всей памяти конкретного юзера."""        return self.memory.get_all(user_id=user_id)3. Интеграция в основного агента: agent_cycle.pyЗдесь показано, как основной цикл бота забирает данные перед генерацией ответа и сохраняет новый опыт после.Pythonfrom openai import OpenAIfrom memory_layer import SharedMemoryLayerclass AutonomousBot:    def __init__(self):        self.llm = OpenAI()        self.memory_module = SharedMemoryLayer()    def process_turn(self, user_id: str, user_input: str) -> str:        # Шаг 1: Субагент подготовки контекста        print("[Log] Извлечение релевантных воспоминаний...")        past_facts = self.memory_module.fetch_relevant_facts(user_input, user_id=user_id)                # Шаг 2: Формирование системного промпта        system_prompt = f"""Ты — интеллектуальный агент-помощник.        Опирайся на следующие известные факты о пользователе для персонализации ответа:        {past_facts}                Если фактов нет, просто отвечай на запрос пользователя.        """        # Шаг 3: Основной цикл выполнения        response = self.llm.chat.completions.create(            model="gpt-4o",            messages=[                {"role": "system", "content": system_prompt},                {"role": "user", "content": user_input}            ]        )        agent_reply = response.choices[0].message.content        # Шаг 4: Асинхронное/фоновое обновление памяти        # (в продакшене это лучше вынести в отдельный поток или очередь)        print("[Log] Сохранение новых фактов в Mem0...")        self.memory_module.store_cycle_context(user_input, agent_reply, user_id=user_id)        return agent_replyif __name__ == "__main__":    bot = AutonomousBot()    user = "alex_dev_01"    print("--- Итерация 1 ---")    query_1 = "Привет! Запомни, что я предпочитаю писать скрипты оптимизации систем на PowerShell, а для изоляции использую Docker."    print("User:", query_1)    print("Bot:", bot.process_turn(user_id=user, user_input=query_1))        print("\n--- Итерация 2 (Проверка памяти) ---")    query_2 = "Как мне лучше автоматизировать глубокую очистку временных файлов в ОС Windows перед созданием образа?"    print("User:", query_2)    # Здесь бот должен вспомнить про PowerShell и Docker благодаря Mem0    print("Bot:", bot.process_turn(user_id=user, user_input=query_2))        print("\n[Debug] Все сохраненные факты в БД:")    for fact in bot.memory_module.debug_memory(user_id=user):        print("-", fact['memory'])Запуск и проверка интеграцииТы можешь быстро протестировать этот каркас через терминал или скрипт в своей среде (код полностью готов для выполнения в Windows PowerShell или внутри подсистемы WSL):PowerShell# 1.
