# Техническое задание на доработку Landing Page «Агентство строительного контроля» ##
1. Общие положения * **Тип проекта:** Доработка существующего Landing Page
(одностраничного сайта). * **Текущий сайт:** `stroitelny-kontrol.ru` * **Макет для
доработок:** Figma. * **Платформа (CMS):** MODX Revolution (верстка интегрируется в
соответствующие чанки). * **Цель сайта:** Привлечение целевого трафика,
информирование об услугах и сбор заявок (лидогенерация) от потенциальных клиентов.
## 2. Требования к дизайну и верстке * **Цветовая схема:** Фирменный синий (основной
акцентный цвет), белый (фон), темно-серый/черный (текст), светло-серый (фоновые
блоки). * **Адаптивность:** Сайт должен корректно отображаться на всех типах устройств.
Сделать адаптив для всех внедряемых изменений. * **Кроссбраузерность:** Поддержка
актуальных версий Google Chrome, Safari, Mozilla Firefox, Yandex Browser, Microsoft Edge. --
- ## 3. Структура страницы и изменения (Поэкранное описание) ### 3.1. Шапка сайта
(Header) * **Элементы:** Логотип компании, контакты (телефон), кнопки «Обратный
звонок» и «Написать нам». * **Текст «Показать номер»:** Должен быть выровнен строго по
центру относительно самого номера телефона. ### 3.2. Первый экран (Hero Section) *
**Фон:** Тематическое затемненное фото (стройка/леса). * **Заголовок (H1):**
«Профессиональные услуги строительного контроля». * **Лид-форма (справа):** Поля для
ввода с учетом новых требований к валидации (см. раздел 4). ### 3.3. CTA-блок и формы
обратной связи в теле страницы * **Верстка для внутренних страниц (CMS):** Должна
находиться в Чанке `call-back`. * **Верстка для главной страницы (CMS):** Должна
находиться в Чанке `page-content-index`. * **Оптимизация для главной:** Форму
необходимо подогнать: фон не нужно делать во всю ширину экрана (лучше ограничить по
ширине контента страницы), заголовок при необходимости можно разместить в две
строчки. ### 3.4. Блок Контакты * **Отступы:** Необходимо скорректировать (уменьшить)
вертикальные отступы между строками «Наш адрес», «Телефоны» и «E-mail», чтобы блок
выглядел более компактно (согласно скриншоту-референсу). ### 3.5. Подвал сайта
(Footer) * **Текст «Показать номер»:** Выровнять по центру относительно номера
телефона (по аналогии с шапкой). * **Кнопка «Написать нам»:** Выровнять строго по
левому краю. * **Нижняя строка:** Копирайт (©), ссылки на «Карту сайта», «Политику
конфиденциальности» и «Политику обработки Cookies». ### 3.6. Плавающие элементы
(Sticky UI) * **Кнопка обратной связи (Левый нижний угол):** Взять функционал по
аналогии с сайтом `up-cds.ru` (иконки звонка и почты), поменяв цвета под текущий дизайн
сайта. * **Уведомление об использовании Cookies (Правый нижний угол):** Плашка с
текстом и ссылкой на политику cookie. При нажатии на кнопку «Я согласен» табличка
должна пропадать. В мобильной версии плашка **не должна** перекрывать кнопку
обратной связи в левом углу. --- ## 4. Функциональные требования ### 4.1. Формы
обратной связи и строгая валидация (по аналогии с `advokat-kachestva.ru`) *
**Обязательные поля:** Обязателен ввод либо Телефона, либо E-mail. Выдавать
соответствующие ошибки при неверном формате ввода или незаполненных обязательных
полях. * **Контроль ввода номера телефона:** * Цифры `+`, `7`, `8` не учитываются при
попытке ввода в качестве первого знака. * Префикс `+7` уже должен быть статично
написан в начале поля. * При клике в любое место поля ввода, курсор должен
автоматически вставать в начало ввода (сразу после `+7`). ### 4.2. Чекбокс согласия на
обработку персональных данных (под кнопкой во всех формах) * **Статус по умолчанию:**
Чекбокс **не стоит**. Пользователь должен поставить его вручную. * **Валидация:** Если
посетитель не поставил чекбокс, ярко визуально указывать на необходимость его
установки при попытке отправки. * **Текст:** «Я ознакомлен(а) с политикой
конфиденциальности и даю согласие на обработку персональных данных.» * **Ссылки:**
Текст должен содержать ссылки на соответствующие страницы документов. Документы
должны открываться в **новом окне браузера**. ### 4.3. Новая форма «Написать нам»
(Модальное окно) * **Дизайн:** Добавить самостоятельно без готового макета. Визуально
сделать такой же, как модальное окно «Обратный звонок». * **Поля:** Имя, Телефон,
Email, Сообщение (добавить 2 дополнительных поля относительно стандартной формы). *
**Файлы:** Добавить кнопку «Прикрепить файл» (подобрать соответствующие иконки
самостоятельно). ### 4.4. Настройка писем и интеграция * Поменять почту получателя
заявок со всех форм. * Поменять заголовки для писем со всех форм, чтобы было понятно,
откуда пришла заявка. * Подключить и протестировать отправку с новой формы
«Написать нам».
